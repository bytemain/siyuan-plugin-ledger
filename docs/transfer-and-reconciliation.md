# 转账与对账

本文档介绍如何记录账户间转账、核对账户余额，以及月末对账的最佳实践。

---

## 一、账户间转账

### 1.1 什么时候用转账

转账用于记录资金在你的不同账户之间流动，不涉及收入或支出。常见场景：

- 银行卡转到支付宝
- 支付宝提现到银行卡
- 活期转存定期
- 还信用卡（详见 [信用卡还款](./credit-card-and-reimbursement.md)）

### 1.2 使用转账弹窗

1. 按 `⇧⌘T`（Mac）或 `⇧Ctrl+T`（Windows），或输入 `/转账`。
2. **转出账户**：选择资金来源（如 `Assets:Bank:Checking`）。
3. **转入账户**：选择资金目的地（如 `Assets:Alipay`）。
4. **金额**：输入转账金额。
5. **备注**（可选）：填写说明（如"充值支付宝"）。
6. 点击记账。

### 1.3 转账的复式记账原理

转账在复式记账中表现为资金从一个资产账户流向另一个资产账户，总资产不变：

```
Assets:Alipay         +1000    ← 支付宝增加
Assets:Bank:Checking  -1000    ← 银行卡减少
```

两边金额之和为 0，这就是"复式记账必须平衡"的核心原则。

## 二、常见转账场景

### 场景 1：银行卡充值支付宝

```
转出：Assets:Bank:Checking（银行卡）
转入：Assets:Alipay（支付宝）
金额：1000
```

### 场景 2：微信提现到银行卡

```
转出：Assets:WeChatPay（微信）
转入：Assets:Bank:Checking（银行卡）
金额：5000
```

### 场景 3：活期转定期

如果你添加了储蓄账户：

```
转出：Assets:Bank:Checking（活期）
转入：Assets:Bank:Savings（储蓄）
金额：10000
```

### 场景 4：还信用卡

还信用卡本质上也是一笔转账。推荐使用专用的 `/还信用卡` 命令，详见 [信用卡还款](./credit-card-and-reimbursement.md#14-还信用卡还款)。

## 三、查看账户余额

### 3.1 侧边栏实时余额

打开侧边栏（`⌥⌘L` / `Alt+Ctrl+L`），可以看到：

- **资产区域：** 每个资产账户的当前余额
- **负债区域：** 各信用卡/花呗的欠款金额（仅显示非零余额）
- **净资产：** 总资产 + 总负债（负债为负值）
- **本月收支：** 当月的收入和支出汇总

> 💡 点击侧边栏的 **刷新** 按钮可更新余额数据。

### 3.2 仪表盘查看

打开仪表盘（`⇧⌘L` / `⇧Ctrl+L`），在月度概览中可以看到当月的收支汇总。

## 四、月末对账

### 4.1 为什么要对账

定期对账可以确保你的记录与实际余额一致，及时发现漏记或错记的交易。建议每月月末进行一次。

### 4.2 对账步骤

#### 步骤 1：核对各账户余额

1. 打开侧边栏，查看插件中各账户的余额。
2. 打开你的银行 APP、支付宝、微信等，查看实际余额。
3. 逐个比对，记录差异。

#### 步骤 2：排查差异

如果某个账户的余额不对，可能的原因：

| 差异情况 | 可能原因 | 解决方式 |
|---------|---------|---------|
| 插件余额偏高 | 有消费忘记记了 | 补录缺失的支出 |
| 插件余额偏低 | 有收入忘记记了 | 补录缺失的收入 |
| 差额刚好是某笔金额 | 某笔交易记错了账户 | 编辑该交易修正账户 |
| 差几毛几分钱 | 舍入误差或小额手续费 | 记一笔调整交易 |

#### 步骤 3：标记已确认的交易

将核实过的交易状态改为 **✓ 已确认（cleared）**，这样下次对账时只需要关注"未确认"的交易即可。

### 4.3 使用期初余额

如果你是从使用一段时间后才开始用插件记账，可以通过 **权益（Equity）** 账户设置期初余额：

1. 使用转账功能。
2. **转出**：`Equity:Opening-Balances`（期初余额）。
3. **转入**：对应的资产账户（如银行卡）。
4. **金额**：填写该账户的实际余额。

这样你的账户余额就和实际一致了。

## 五、多币种转账

如果涉及不同币种的转账（如人民币兑换美元），每个过账行（posting）可以指定不同的币种。插件会按币种分别计算余额。
