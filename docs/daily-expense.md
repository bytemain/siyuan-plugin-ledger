# 日常记账场景

本文档汇总了最常见的日常记账场景，帮助你快速找到对应的记录方式。

---

## 一、基本支出记录

### 场景：日常消费（餐饮、交通、购物……）

**快记模式**（推荐日常使用）：

```
午饭 25                    → 默认账户扣款
咖啡 35 微信               → 微信支付
打车 32 支付宝             → 支付宝支付
超市 186 现金              → 现金支付
电影票 80 银行卡            → 银行卡支付
```

**弹窗模式**（需要更多细节时）：

1. 按 `⇧⌘E` 打开支出弹窗。
2. 填写金额、收款方（如"海底捞"）。
3. 选择支出分类（如 `Expenses:Food:Dining`）。
4. 选择付款账户。
5. 可选填写备注和标签。
6. 点击记账。

> 💡 **智能推荐：** 输入收款方时，插件会自动补全历史收款方，并根据上次记录自动选择分类和金额。比如你上次在"海底捞"消费 258 元用的微信，下次输入"海底捞"时会自动推荐同样的分类和金额。

### 场景：带日期的补录

忘记记账了？可以在快记中指定日期：

```
2025-03-15 午饭 25 微信
2025-03-10 加油 400 银行卡
```

支持的日期格式：`YYYY-MM-DD` 和 `YYYY/MM/DD`。

### 场景：添加标签备注

标签可以帮助你筛选和分类交易：

```
打车去客户 86 微信 标签:待报销
团建聚餐 500 信用卡 标签:团建
```

弹窗模式中也有**标签**输入栏，支持自动补全历史标签。

## 二、收入记录

### 场景：发工资

1. 按 `⇧⌘I` 或输入 `/收入` 打开收入弹窗。
2. **金额**：输入到账金额（如 `15000`）。
3. **收入来源**：选择 `Income:Salary`（工资）。
4. **转入账户**：选择收款的银行卡。
5. 点击记账。

### 场景：收到奖金

操作同上，**收入来源** 改为 `Income:Bonus`（奖金）。

### 场景：投资收益

- 股票分红：收入来源选 `Income:Investment:Dividends`
- 利息收入：收入来源选 `Income:Investment:Interest`

### 场景：收到报销

详见 [信用卡与报销指南](./credit-card-and-reimbursement.md) 的报销部分。快记方式：

```
报销 差旅费 380
报销 打车费 86 银行卡
```

## 三、不同支付方式

快记模式支持多种支付方式别名，你可以直接用日常叫法：

| 你说的 | 插件理解为 |
|-------|----------|
| `微信` 或 `wechat` | Assets:WeChatPay |
| `支付宝` 或 `alipay` | Assets:Alipay |
| `现金` 或 `cash` | Assets:Cash |
| `银行卡` | Assets:Bank:Checking |
| `信用卡` 或 `credit` | Liabilities:CreditCard:CMB |
| `招行信用卡` 或 `招行` | Liabilities:CreditCard:CMB |
| `工行信用卡` 或 `工行` | Liabilities:CreditCard:ICBC |
| `花呗` | Liabilities:HuaBei |
| `借呗` | Liabilities:JieBei |

> 💡 不指定支付方式时，会使用你在设置中配置的**默认支付账户**。

## 四、分账记录

### 场景：一笔消费涉及多个分类

比如去超市买了食材和日用品，想分别记到不同分类：

1. 按 `⇧⌘E` 打开支出弹窗。
2. 勾选 **分账** 选项。
3. 添加多行分录：
   - `Expenses:Food:Groceries` — 食材 ¥150
   - `Expenses:Shopping:Daily` — 日用品 ¥86
4. 选择付款账户（如支付宝）。
5. 点击记账。

插件会自动生成一笔包含多个分类的复式记录，总金额从付款账户扣除。

## 五、编辑和删除已有账目

### 编辑交易

在笔记中找到已记录的交易块，**右键点击** → 选择 **✏️ 编辑交易**，即可在弹窗中修改金额、分类等信息。

也可以在仪表盘的最近交易列表中点击某笔交易进行编辑。

### 删除交易

右键点击交易块 → 选择 **🗑️ 删除交易**，确认后即可删除。

## 六、交易状态

每笔交易可以有三种状态：

| 状态 | 标记 | 适用场景 |
|------|------|---------|
| ✓ 已确认 | `cleared` | 已对账确认的交易 |
| ? 待处理 | `pending` | 等待到账或确认中 |
| ~ 未确认 | `uncleared` | 刚记录、尚未核实 |

在弹窗中可以选择交易状态，默认为"未确认"。

> 💡 **建议工作流：** 日常记录时保持默认状态，月末对账时将核实过的交易改为"已确认"。这样可以清楚地知道哪些交易已经核对过。

## 七、一天记账示例

下面是一个典型工作日的记账流程，全部使用快记模式：

```
早餐 12                         → 默认账户
午饭 35 微信                     → 微信支付
地铁 6 支付宝                    → 支付宝
咖啡 28                         → 默认账户
晚饭 超市买菜 45 微信             → 微信支付
打车回家 25 微信                  → 微信支付
```

每条只需几秒钟，一天的账目轻松搞定。
